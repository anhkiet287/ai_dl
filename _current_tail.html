Vision preprocessing

Image preprocessing

All image normalization and resizing are delegated to CLIP's official processor to stay aligned with the pretrained checkpoint.

Inside N24NewsCLIPDataset

  • Read image with PIL
  • Convert to RGB
  • Call processor(images=image, return_tensors="pt")
  • Squeeze to a single pixel_values tensor

Why use CLIPProcessor

  • Applies the same resize / crop / rescale / normalize recipe used during CLIP pretraining
  • Produces the exact tensor shape expected by the frozen vision encoder
  • Reduces preprocessing bugs compared with hand-written transforms

Tokenization details

  • Tokenizer: CLIP tokenizer via CLIPProcessor
  • Padding: enabled
  • Truncation: enabled
  • Maximum length: 77 tokens
  • Output: class-level text embeddings in the shared 512-d CLIP space
CLIPProcessor return keys
- pixel_values: image tensor batch (C, H, W), resized & normalized
- input_ids: tokenized text ids
- attention_mask: 1 for real tokens, 0 for padding

Notes
- Wraps CLIPImageProcessor + CLIPTokenizer
- Returns PyTorch / TensorFlow tensors when requested; lists otherwise
Batch assembly

DataLoader and collate function

The loader is designed to be robust against image read failures while keeping batch metadata available for later analysis.

Standard loaders

  • Batch size: 32
  • Shuffle: train = true, val/test = false
  • num_workers: 0
  • pin_memory: false
  • drop_last: false

clip_collate_fn

The custom collate function filters out None samples and returns a structured batch dictionary.

{ "pixel_values": [B, 3, 224, 224], "label_id": [B], "image_id": list[str], "image_path": list[str], "section": list[str] }
  • Filters None samples before stacking.
  • Keeps ids/paths/section strings for analysis and error tracing.
Few-shot protocol

Few-shot sampler

Few-shot training uses a separate loader that samples exactly k examples per class from the training split.

How the sampler works

  1. Group train_df by label_id.
  2. Sample min(shot, len(class_subset)) examples per class.
  3. Reset the index and rebuild a dataset from the sampled rows.
  4. Create a new DataLoader with batch_size = 16.

Main experiment setting

  • Main comparison shot: 10 examples per class
  • Few-shot batch size: 16
  • Shot-scaling study: 1, 4, and 10 shots
  • Reason: isolate sample efficiency under limited supervision
