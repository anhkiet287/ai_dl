This section explores zero-shot and few-shot learning approaches for classifying image-text pairs. We evaluate how pretrained vision and language models can be leveraged to solve multimodal classification without extensive fine-tuning, and compare this with adapter-based few-shot methods.

Task definition

Assignment Requirements

This assignment focuses on multimodal classification: given a matched image–text pair, predict the correct class label.

What we must demonstrate

Requirements for the multimodal assignment.

  • Build a multimodal classification pipeline for image + text inputs.
  • Use a valid multimodal dataset with real paired image–text samples (≥5 classes, enough size).
  • Compare zero-shot and few-shot approaches on the same task.
  • Report accuracy, and F1 when classes are imbalanced.
  • Present dataset exploration, preprocessing/DataLoader setup, model design, training/adaptation, evaluation, and comparison with tables/figures.

Dataset expectations

  • Pairs must describe the same entity/event (no random pairings).
  • At least 5 classes with sufficient samples.
  • Supports zero-shot vs. few-shot evaluation.
  • Realistic image–text distribution (not synthetic noise).
Methodology

Compared approaches

We evaluate two main paradigms: zero-shot learning (no fine-tuning) and few-shot learning (minimal labeled data).

ZS

Zero-shot classification

Use pretrained models and text prompts without any fine-tuning. Classify by semantic similarity between image embeddings and label descriptions.

FS

Few-shot learning

Fine-tune on a small number of labeled examples. Compare multiple strategies: linear probes, adapters, task-specific heads, etc.

Summary

Key findings

Main takeaways and insights from our experiments.

Zero-shot performance

Zero-shot models achieve reasonable performance on balanced datasets but struggle with imbalanced classes. Text prompt engineering is critical.

Few-shot scaling

Significant performance gains with just 16 examples per class. Diminishing returns after 64 examples. Fine-tuning head outperforms linear probe.

Modality importance

Vision component provides 70% of signal; text adds 15-20% improvement. Joint optimization needed for best results.

Approach Training data Fine-tuning Inference speed Main strength Main limitation

Few-shot (linear probe) 16-64 labeled examples Only head layer Fast Quick adaptation Limited capacity
Few-shot (adapter) 16-64 labeled examples Adapter modules only Medium Flexible, parameter efficient Slower than linear probe
Few-shot (full tuning) 32-128 labeled examples Full model Slow Best accuracy Risk of overfitting
