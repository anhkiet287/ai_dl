Zero-shot learning - no fine-tuning

Zero-shot Classification

Zero-shot learning classifies images without any task-specific fine-tuning. We use pretrained vision-language models (e.g., CLIP) to compute semantic similarity between image embeddings and textual label descriptions, then select the highest-scoring label.

Motivation

Why zero-shot learning?

Understanding the practical appeal and limitations of zero-shot approaches.

Key advantages

  • No labeled data required
  • No training time overhead
  • Leverages general pretrained models
  • Fast deployment and inference
  • Works on novel classes

Practical use cases

  • Rapid prototyping
  • New dataset / domain exploration
  • Baseline establishment
  • Resource-constrained settings
  • One-shot or few-label scenarios
Methodology

Zero-shot pipeline

Step-by-step breakdown of the zero-shot classification approach.

1

Select pretrained model

Choose a vision-language model (e.g., CLIP, ALIGN, BEiT+BERT). Model should be pretrained on large-scale image-text data.

2

Create label descriptions

For each class label, write one or more textual descriptions (e.g., "A photo of a cat", "A dangerous object", "High-quality product").

3

Encode labels

Use the model's text encoder to generate embeddings for all label descriptions. Store these embeddings for efficient comparison.

4

Encode test images

Use the model's vision encoder to generate embeddings for test images. No labels needed—only the images are processed.

5

Compute similarity

Compute similarity scores between each image embedding and all label embeddings (typically cosine similarity or dot product after normalization).

6

Predict & output

Select the label with the highest similarity score as the prediction. Optionally, apply softmax to get confidence scores.

Text design

Label description strategy

How we construct the text prompts for effective zero-shot classification.

Prompt design principles

  • Be descriptive but concise
  • Use context (e.g., "A photo of...")
  • Include domain keywords
  • Test multiple prompt variations
  • Ensure consistency across labels

Template approaches

  • Simple: "class_name"
  • Context: "A photo of a class_name"
  • Detailed: "An image of a class_name in a ..."
  • Multiple variants per class
  • Ensemble predictions across variants

Example prompt template for your dataset

# For each class, construct a description:
descriptions = {
  "class_A": "A photo of class_A",

  ...
}

# Or use multiple prompts per class:

  "class_A": ["A photo of class_A", "An image of a class_A", "class_A object"],
  ...

Analysis

Strengths & limitations

When zero-shot works well and where it struggles.

✓ Strengths

  • No labeled data needed
  • Fast inference
  • Works on novel classes
  • Easy to implement
  • Interpretable decisions

✗ Limitations

  • Prompt engineering required
  • Lower accuracy than fine-tuned
  • Fails on out-of-distribution data
  • Language bias from pretraining
  • Domain mismatch issues

When zero-shot succeeds

Zero-shot performs well when:

  • Dataset domain is well-covered by pretraining
  • Class descriptions are clear and unambiguous
  • Image-text pairs are aligned and informative
  • Classes are visually or semantically distinct
  • Inference speed is prioritized over accuracy

When zero-shot struggles

Zero-shot struggles when:

  • Fine-grained or domain-specific classification needed
  • Visual domain is far from pretraining data
  • Classes are semantically similar
  • Text is uninformative or misleading
  • High accuracy is required
